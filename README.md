## 📚 Estructura de Asignaciones y Proyectos

El repositorio está estructurado en fases modulares que demuestran la evolución técnica desde la maquetación nativa hasta arquitecturas avanzadas basadas en componentes SPA:

### ⚽ [Fase 01: Merchandising](./fase01-merchandising/index.html)
* **Descripción:** Tienda interactiva para la campaña "Colección Copa 2026".
* **Enfoque Técnico:** Estructuración semántica en HTML5, layouts modernos con CSS Grid/Flexbox y renderizado dinámico inicial de productos.

### 🎛️ [Fase 02: Arquitectura](./fase02-arquitecturamundial/index.html)
* **Descripción:** Módulo avanzado de gestión interna para control de stock.
* **Enfoque Técnico:** Desacoplamiento lógico utilizando módulos nativos de JavaScript (ES Modules), algoritmos de validación reactiva de datos y control de flujos con lógica condicional.

### 🌱 [Fase 03: Producto MVP](./fase03-producto-mvp/dist/index.html)
* **Descripción:** Producto Mínimo Viable (MVP) enfocado en la automatización de problemas del entorno real.
* **Enfoque Técnico:** Migración del entorno de desarrollo a **Vite + React**. Creación de hooks iniciales, manejo de interfaces SPA y optimización de código para producción.

### 🚀 [Fase 04: Luxury Showroom](./fase04-despliegue-final/dist/index.html)
* **Descripción:** Catálogo de presentación premium consolidado.
* **Enfoque Técnico:** Configuración de compilación optimizada en Vite, empaquetado para distribución en la web (build) y despliegue continuo de entornos estáticos.

### 🛍️ [Fase 05: Fake Store API](./fase05-tiendaApi/dist/index.html)
* **Descripción:** Catálogo dinámico síncrono conectado a una base de datos externa.
* **Enfoque Técnico:** Implementación de arquitectura avanzada de software dividida en **Componentes Inteligentes (Contenedores Lógicos)** y **Componentes Puros de Presentación**. Consumo asíncrono de servicios mediante Fetch API a `fakestoreapi.com`, control de estados condicionales (`loading`, `error`, `success`) y diseño estético futurista con *Glassmorphism*.
